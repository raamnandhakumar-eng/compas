# Data sources and stimulus provenance

## Occupations

The planned live audit uses eight synthetic occupations: four frontline or operational roles and four knowledge-work roles. Selection uses O*NET tasks and skills, May 2025 BLS OEWS employment and wage estimates, and broad education-pattern considerations.

The machine-readable registry is [`data/occupations/occupation_registry.csv`](../data/occupations/occupation_registry.csv). It records SOC code, title, broad group, median wage, employment, typical education, O*NET version, BLS year, selection reason, and source URL.

O*NET 30.3 is used as the occupational reference release. Resume wording is newly written and does not reproduce complete O*NET task descriptions.

## Perceived name signals

Names are experimental **perceived name signals**, not verified demographic identities.

### 2020 Census name files

The source screen uses complete published tables for names occurring approximately 100 or more times:

- first names by race and Hispanic origin;
- first names by sex;
- last names by race and Hispanic origin.

The files provide national aggregate counts and percentages. Census counts include disclosure-protection adjustments, and the research versions may contain negative values generated by the disclosure-avoidance process. The data do not identify individuals or establish the identity of anyone with a given name.

Official source page:

- https://www.census.gov/topics/population/genealogy/data/2020_names.html

### Social Security Administration name files

SSA national baby-name files provide first-name counts by birth year. They are used to avoid extremely rare first names and to document frequency over the locked source period.

- https://www.ssa.gov/oact/babynames/limits.html

### Current source registry

The eight selected names are stored in `data/name_validation/name_candidates.csv`, with two names per signal group. The file includes first and last name, Census counts and shares, SSA frequency, source year, source file, and screening status.

Every row is marked `source_screened_pending_pretest`. This means the public-source fields are populated but no name has passed the human perception pretest.

### Perception pretest

Approximately 100 to 200 respondents must rate perceived race or ethnicity, perceived gender, familiarity, perceived socioeconomic status, confidence, and unusualness. Only names that pass the locked agreement and balance rules may be used for demographic interpretation in a live audit.

See [`docs/name_signal_validation_protocol.md`](name_signal_validation_protocol.md).

## Synthetic resumes

The repository contains 32 base profiles: four profiles for each of eight occupations. Employer histories and achievements vary across profiles, while all qualifications remain fixed within each matched treatment set.

## Placebo outcomes

`results/placebo/` contains the original 1,280 outcomes from `mock-auditor-v2`. They validate the software and estimator and are not observations from Claude, recruiters, employers, or real applicants.

The expanded confirmatory infrastructure uses `mock-auditor-v3` to test the 512-resume and 2,560-evaluation design before any paid live run.

## Reproducibility date

Public-source mappings and links were checked on July 23, 2026. Any live audit must store the exact model identifier, API version, date, prompt version, full prompts, temperature, trial number, raw response, parser status, and error type.
